使用配置

下面是页面上本身的一组原生的radio，注意观察第3、4、8的属性,美化的结果如上面所示。

需要注意的是，组件里与radio对应的label默认是通过dom.next()获取的，如果获取不到，需要传递getLabelFunc来获取label的位置

使用radio组件美化，只需要使用如下方式即可

        //初始化
        var r = new radio('#J_Content input',{
            hasLabel:true, //是否需要label-for，默认是false
            accessible:true//是否需要无障碍，默认为false
        })
        //执行
        r.render();

如果不需要提供默认的样式，可以自己提供样式地址只需要在初始化的时候，提供配置的样式地址即可，然后使用自己自定义的样式，需要给这些默认的样式名提供ks-radio(默认样式),ks-radio-checked(选中),ks-radio-disabled(禁用),ks-radio-hover(hover)样式

        //初始化
        var r = new radio('#J_Content input',{isUseCss:false});

如果页面的结构通过radio 和label 不在同一级等复杂一些的情况，初始化的时候可以传入一个函数进行关联

        //首先定义一个查找label节点的函数
        function getLabel(value){
            return value.next('label');
        }
        //初始化传入getLabel
         var r = new radio('#J_Content input',{
            hasLabel:true,
            accessible:true,
            getLabelFunc:getLabel //传入查找函数
        })

radio提供的外部方法

1、根据index禁用radio



        //禁用
        r.setDisabled(index);
        //恢复
        r.setDisabled(index);

2、获取所有选中的radio的索引



        //返回数字数组
        r.getAllChecked();

3、还原成为原生的radio



        //还原成为原生的radio
        r.recoverradio();
